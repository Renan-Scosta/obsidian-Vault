# O que é?


A API REST é um estilo arquitetura; que utiliza métodos HTTP padrão (GET, POST, PUT, DELETE) para interagir com recursos representados por [[URIs]]. É sem estado, exigindo informações completas da requisição, utiliza códigos de status HTTP e normalmente se comunica via JSON/XML. popular por sua simplicidade e escalabilidade.

![[Pasted image 20260225110849.png]]


Interface de programação de aplicação (**API**, do inglês application Programing Interface)

- API e o browser se comunicam através de mensagens HTTP
- Existem dois tipos de mensagens HTTP, requests e responses, cada uma com seu próprio formato.


**REST** é o acrônimo de REpresentation State Transfer. Esse padrão é descrito em seis restrições.

- [[Uniform Interface]]
- [[Stateless]]
- [[Cacheable]]
- [[Client-Server]]
- Code on Demand (opcional)


- **RESTFul** é o termo utilizado para descrever APIs HTTP que adotam o padrão REST

- Pode se dizer que é uma "ponte" entre o cliente (que pode ser o browser com a aplicação frontend ou até mesmo outra API/sistema) e o servidor de banco de dados.
